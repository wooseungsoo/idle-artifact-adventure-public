Idle Artifact Adventure (Public Portfolio)
프로젝트 개요

장르: 방치형 RPG

플랫폼: PC / Mobile

엔진: Unity

상태: 프로토타입 (핵심 플레이 루프 검증 완료)

본 프로젝트는 방치형 RPG 장르의 핵심 시스템 구조와 데이터 설계 방식을 검증하기 위해 제작한 개인 포트폴리오용 프로젝트입니다.
방치형 게임의 성장 구조, 자동 전투 흐름, 데이터 기반 설계를 중점적으로 구현했습니다.

기획 의도

방치형 RPG의 재미는 “조작”보다 성장 선택과 보상 피드백에 있다고 판단했습니다.
이에 따라 본 프로젝트는 다음 질문에 대한 답을 목표로 설계되었습니다.

최소 조작 환경에서도 플레이어가 성장에 개입하고 있다고 느낄 수 있는가?

자동 전투 구조에서 캐릭터·스킬·아이템의 차별화가 가능한가?

라이브 서비스 확장을 고려한 데이터 구조가 가능한가?

주요 특징
1. 방치형 전투 시스템

영웅 자동 배치 및 자동 전투 진행

스테이지 단위 전투 구조

몬스터 자동 생성 및 반복 전투 루프

2. 성장 시스템

영웅 레벨업 (경험치 기반)

장비 장착 및 강화

스킬 / 특성(Talent) 성장 구조

아이템 합성 및 분해 시스템

3. 튜토리얼 구현

게임 진입 시 핵심 시스템을 안내하는 튜토리얼 플로우 구현

영상 내에서 실제 튜토리얼 동작 확인 가능

신규 유저의 시스템 이해를 돕는 구조 설계

데이터 관리 (CSV / JSON)

영웅, 스킬, 아이템 등 주요 데이터는 CSV / JSON 기반으로 분리 설계

코드 수정 없이 데이터 조정이 가능하도록 구성

밸런싱 및 콘텐츠 확장을 고려한 구조

프로토타입 단계에서는 데이터 구조 검증을 우선하여
최소 단위의 데이터만 구성했습니다.

기술적 구현 포인트
FSM (Finite State Machine)

영웅 및 몬스터 자동 전투 상태 관리

전투 흐름을 상태 단위로 분리하여 가독성과 확장성 확보

Object Pooling

몬스터, 이펙트, UI 오브젝트 반복 생성/제거 최적화

전투 반복 구조에서의 성능 안정성 확보

이벤트 중심 구조

Observer 패턴 기반 EventManager 설계

시스템 간 결합도를 낮춘 구조

Dictionary 기반 데이터 관리

대량 데이터 검색 및 접근 성능 고려

아이템/스킬 ID 기반 참조 구조

클라이언트 구조 요약

Manager 구조

GameManager: 게임 전체 흐름 및 상태 관리

GameDataManager: 데이터 로딩/저장 관리

DungeonManager: 전투 스테이지 관리

HeroManager: 영웅 데이터 및 상태 관리

UIManager: 전체 UI 제어

EventManager: 게임 이벤트 중앙 관리

레퍼런스 및 분석

본 프로젝트는 방치형 RPG 장르 분석 과정에서
Eternal Ember를 주요 레퍼런스로 삼아 핵심 플레이 루프를 분석하였으며,
해당 구조를 바탕으로 시스템을 재해석하여 구현했습니다.

미구현 / 확장 계획

후반 콘텐츠 확장 (보스 레이드, 챌린지 던전 등 PvE 모드 다양화)

Firebase 기반 경매장 시스템

파밍 아이템 거래

동시성/중복 처리 고려한 구조 설계 예정

라이브 서비스 운영을 고려한 경제 밸런스 확장
