# Idle Artifact Adventure (Prototype)

Unity로 제작한 **방치형 RPG 프로토타입**입니다.  
본 저장소는 채용/리뷰 목적의 **시스템 구조 및 기획 의도 공유**를 위해 정리한 Public 버전이며,  
일부 아트/사운드 에셋은 라이선스 문제로 제외되어 있습니다.

---

## 프로젝트 목적
- 방치형 RPG의 핵심 루프 검증: **성장 → 자동 전투 → 보상 → 성장**
- 플레이어/히어로 데이터 관리 및 전투 수치 계산 흐름 정리
- 스킬/특성 구조를 구현하여 **콘텐츠 확장 가능한 형태**로 구성

---

## 핵심 게임 루프
1. 플레이어 및 히어로 능력치 계산
2. 자동 전투 진행 (스킬/특성 반영)
3. 보상 획득
4. 성장 요소 반영 후 다음 전투로 반복

---

## 시스템 구성 요약

### 1) 데이터 / 스탯 관리
- 플레이어 및 히어로 능력치를 중앙 로직에서 관리하여 계산 규칙이 분산되지 않도록 구성했습니다.
- 전투 결과가 성장(재화/강화)으로 환류되는 흐름을 명확히 두었습니다.

### 2) 스킬 / 특성
- 스킬/특성은 ID 및 조건 기반 처리로 구성했습니다.
- 신규 스킬/특성 추가 시 기존 전투 흐름을 크게 변경하지 않도록 설계했습니다.

### 3) 성능 고려
- Coroutine 기반 업데이트
- 오브젝트 풀링을 활용한 성능 안정화

---

## 담당 역할
- 플레이어 / 히어로 데이터 관리
- 스킬 / 특성 시스템 구현
- 전투 수치 계산 및 타겟팅 로직 구성
- 이펙트/사운드 연동 (본 Public 레포에서는 일부 리소스 제외)

---

## 저장소 구성
- `Assets/Scripts/` : 핵심 시스템 및 게임 로직
- `Assets/Scenes/MainScene_v2.unity` : 최종 통합 씬  
  - 에셋 제외로 일부 참조(Missing)가 발생할 수 있습니다.
- `Assets/Prefabs/` : 논리 구조 중심 프리팹

---

## 한계 및 향후 개선 방향
- 프로토타입 단계로 UX/UI 폴리싱은 미완료입니다.
- 밸런싱 작업 효율화를 위해 데이터 구조를 ScriptableObject 또는 테이블 기반으로 전환하는 개선을 고려하고 있습니다.

---

## 리뷰 가이드
- 코드 리뷰는 `Assets/Scripts/` 폴더부터 보시면 핵심 루프/구조 파악이 빠릅니다.
- UI 흐름/수치 설계/기획 의도 상세는 포트폴리오 PDF에서 보완합니다.
